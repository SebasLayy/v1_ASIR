Autenticación centralizada, almacenamiento compartido y gestor de colas para cómputo de alto rendimiento

🎯 ¿Qué vamos a construir?

Un clúster de cómputo formado por 4 máquinas virtuales donde:

  • LDAP centraliza los usuarios (sebas1, sebas2, sebas3) y el grupo outes.
  • NFS comparte los directorios /home y /opt/apps desde el servidor.
  • autofs monta automáticamente los homes en los nodos de cómputo.
  • Slurm gestiona trabajos en dos particiones: outes1 (nodo1) y outes2 (nodos 2 y 3).
  • Environment Modules permite cargar entornos de software dinámicamente.
✅ Esta infraestructura es típica en centros de supercomputación: gestión unificada de usuarios, almacenamiento compartido y planificación de tareas.

🏛️ Arquitectura final (clúster con 4 nodos)

[PC anfitrión] → SSH puerto 2222 → Computer-0-0 (10.0.2.10) [LDAP + NFS + slurmctld] │ ┌────────────┼────────────┐ ↓ ↓ ↓ computer-0-1 computer-0-2 computer-0-3 172.16.0.11 172.16.0.12 172.16.0.13 cliente LDAP cliente LDAP cliente LDAP autofs NFS autofs NFS autofs NFS slurmd slurmd slurmd Dominio LDAP: ldap.sebasoutes.local Grupo: outes (gid 501) Usuarios: sebas1, sebas2, sebas3 Particiones Slurm: outes1 → computer-0-1 , outes2 → computer-0-2 y computer-0-3
🔗 Todos los nodos tienen internet a través de Computer-0-0 (enrutamiento NAT). Los homes y /opt/apps se comparten por NFS.

🌐 1. Red NAT en VirtualBox y clones enlazados

  • Red NAT: nombre NatLDAP, prefijo 10.0.2.0/24, DHCP deshabilitado.
  • Reenvío de puertos: TCP, puerto anfitrión 2222 → IP invitado 10.0.2.10:22.
  • VM base: Ubuntu Server 24.04 (2CPU, 2GB RAM, 200GB disco), usuario soutes, OpenSSH instalado.

Clones enlazados

  • Computer-0-0 : NAT + red interna "intnet" → IP 172.16.0.10/16
  • computer-0-1 : solo red interna → IP 172.16.0.11/16
  • Clonar computer-0-1 → computer-0-2 (172.16.0.12) y computer-0-3 (172.16.0.13)

🖧 2. Configuración de IPs estáticas (netplan) y enrutamiento

Computer-0-0 (controlador)

network: ethernets: enp0s3: dhcp4: no addresses: [10.0.2.10/24] routes: [ { to: default, via: 10.0.2.1 } ] nameservers: { addresses: [8.8.8.8] } enp0s8: dhcp4: no addresses: [172.16.0.10/16] version: 2

Nodos compute (0-1,0-2,0-3) - cada uno con su IP

network: ethernets: enp0s3: dhcp4: no addresses: [172.16.0.11/16] # .12 y .13 para los otros routes: [ { to: default, via: 172.16.0.10 } ] nameservers: { addresses: [8.8.8.8] } version: 2

Habilitar forwarding y NAT (en Computer-0-0)

sudo apt install iptables-persistent -y sudo iptables -t nat -A POSTROUTING -o enp0s3 -j MASQUERADE sudo iptables -A FORWARD -i enp0s8 -j ACCEPT sudo netfilter-persistent save echo "net.ipv4.ip_forward=1" | sudo tee -a /etc/sysctl.conf sudo sysctl -p
✔️ Comprobar: desde computer-0-1 ping google.com → debe funcionar.

🔑 3. SSH sin contraseña entre nodos (root)

# En TODOS los nodos: sudo passwd root # contraseña: alumno sudo sed -i 's/#PermitRootLogin.*/PermitRootLogin yes/' /etc/ssh/sshd_config sudo systemctl restart ssh # En Computer-0-0 como root: ssh-keygen -t rsa for ip in 172.16.0.10 172.16.0.11 172.16.0.12 172.16.0.13; do ssh-copy-id root@$ip done

Añadir entradas en /etc/hosts de todas las máquinas:

172.16.0.10 Computer-0-0 ldap.sebasoutes.local 172.16.0.11 computer-0-1 172.16.0.12 computer-0-2 172.16.0.13 computer-0-3

📁 4. Servidor LDAP (Computer-0-0) y clientes

sudo apt install slapd ldap-utils -y sudo dpkg-reconfigure slapd
  • Dominio DNS: ldap.sebasoutes.local
  • Organización: sebasoutes · Contraseña admin: alumno
  • Backend: MDB · No eliminar al purgar

Crear estructura, grupo outes y usuarios sebas1,2,3

# OUs cat > estructura.ldif << EOF dn: ou=usuarios,dc=ldap,dc=sebasoutes,dc=local objectClass: organizationalUnit ou: usuarios dn: ou=grupos,dc=ldap,dc=sebasoutes,dc=local objectClass: organizationalUnit ou: grupos EOF ldapadd -x -D cn=admin,dc=ldap,dc=sebasoutes,dc=local -W -f estructura.ldif # Grupo outes (gid 501) cat > grupo.ldif << EOF dn: cn=outes,ou=grupos,dc=ldap,dc=sebasoutes,dc=local objectClass: posixGroup cn: outes gidNumber: 501 EOF ldapadd -x -D cn=admin,dc=ldap,dc=sebasoutes,dc=local -W -f grupo.ldif # Usuarios (uidNumber 1010,1011,1012) con home y contraseña "sebas1", etc. ldapadd -x -D cn=admin,dc=ldap,dc=sebasoutes,dc=local -W -f usuarios.ldif

Clientes LDAP (Computer-0-0 y todos los nodos)

sudo apt install libnss-ldap libpam-ldap ldap-utils nscd -y # URI: ldap://172.16.0.10 , base: dc=ldap,dc=sebasoutes,dc=local sudo nano /etc/nsswitch.conf # passwd: files ldap , group: files ldap sudo pam-auth-update # marcar "Create home directory on login" sudo systemctl restart nscd getent passwd | grep sebas # sebas1,sebas2,sebas3 visibles

💾 5. NFS Server (Computer-0-0) + autofs en clientes

sudo apt install nfs-kernel-server -y # Crear los homes si no existen su sebas1; exit; su sebas2; exit; su sebas3; exit sudo mkdir -p /opt/apps echo "/home 172.16.0.0/16(rw,sync,no_subtree_check,no_root_squash)" | sudo tee -a /etc/exports echo "/opt/apps 172.16.0.0/16(rw,sync,no_subtree_check,no_root_squash)" | sudo tee -a /etc/exports sudo exportfs -a && sudo systemctl restart nfs-kernel-server

autofs en cada nodo (0-1,0-2,0-3)

sudo apt install autofs -y echo "/home /etc/auto.home" | sudo tee -a /etc/auto.master echo "/opt/apps /etc/auto.apps" | sudo tee -a /etc/auto.master echo "* 172.16.0.10:/home/&" | sudo tee /etc/auto.home echo "apps 172.16.0.10:/opt/apps" | sudo tee /etc/auto.apps sudo systemctl restart autofs
⚠️ Si autofs no monta /opt/apps, solución rápida: añadir a /etc/fstab 172.16.0.10:/opt/apps /opt/apps nfs defaults 0 0 y hacer mount -a.

⏱️ 6. Sincronización horaria (indispensable para Munge/Slurm)

# En todos los nodos sudo apt install chrony -y sudo systemctl enable chrony --now sudo chronyc makestep
🔧 El error "Header lengths longer than data" se debe a relojes desincronizados o clave munge diferente. Chrony lo resuelve.

⚙️ 7. Slurm: controlador + slurmd + Munge

Munge (clave común)

# Computer-0-0 sudo systemctl stop munge sudo rm -f /etc/munge/munge.key sudo dd if=/dev/urandom bs=1 count=1024 of=/etc/munge/munge.key sudo chmod 400 /etc/munge/munge.key && sudo chown munge:munge /etc/munge/munge.key for i in 1 2 3; do sudo scp /etc/munge/munge.key root@172.16.0.1$i:/etc/munge/ ssh root@172.16.0.1$i "chown munge:munge /etc/munge/munge.key && chmod 400 /etc/munge/munge.key && systemctl restart munge" done sudo systemctl start munge

slurm.conf (corregido con SlurmUser y DebugFlags)

sudo tee /etc/slurm/slurm.conf << 'EOF' ClusterName=cluster SlurmctldHost=Computer-0-0 AuthType=auth/munge SlurmUser=slurm DebugFlags=NO_CONF_HASH MpiDefault=none ProctrackType=proctrack/pgid ReturnToService=2 TaskPlugin=task/none SlurmctldPidFile=/var/run/slurmctld.pid SlurmdPidFile=/var/run/slurmd.pid SlurmctldLogFile=/var/log/slurm/slurmctld.log SlurmdLogFile=/var/log/slurm/slurmd.log StateSaveLocation=/var/spool/slurmctld SlurmdSpoolDir=/var/spool/slurmd SchedulerType=sched/backfill SelectType=select/linear NodeName=computer-0-1 NodeAddr=172.16.0.11 CPUs=2 Sockets=2 CoresPerSocket=1 ThreadsPerCore=1 RealMemory=1900 State=UNKNOWN NodeName=computer-0-2 NodeAddr=172.16.0.12 CPUs=2 Sockets=2 CoresPerSocket=1 ThreadsPerCore=1 RealMemory=1900 State=UNKNOWN NodeName=computer-0-3 NodeAddr=172.16.0.13 CPUs=2 Sockets=2 CoresPerSocket=1 ThreadsPerCore=1 RealMemory=1900 State=UNKNOWN PartitionName=outes1 Nodes=computer-0-1 Default=YES MaxTime=INFINITE State=UP PartitionName=outes2 Nodes=computer-0-2,computer-0-3 MaxTime=INFINITE State=UP EOF

Distribuir configuración y arrancar servicios:

for i in 1 2 3; do scp /etc/slurm/slurm.conf root@172.16.0.1$i:/etc/slurm/ ssh root@172.16.0.1$i "systemctl restart munge && systemctl restart slurmd" done sudo systemctl restart slurmctld sinfo # debe mostrar outes1 y outes2 con nodos idle
🔄 Si los nodos aparecen en "completing*", limpiar spool: sudo rm -rf /var/spool/slurmctld/* ; for i in 1 2 3; do ssh root@172.16.0.1$i "rm -rf /var/spool/slurmd/* && systemctl restart slurmd"; done; sudo systemctl restart slurmctld

📦 8. Environment Modules + ejecutable de prueba

sudo apt install environment-modules -y source /etc/profile.d/modules.sh sudo mkdir -p /opt/apps/testbin sudo tee /opt/apps/testbin/test.x << 'EOF' #!/bin/bash echo "hola $(whoami)" date EOF sudo chmod a+x /opt/apps/testbin/test.x sudo mkdir -p /usr/share/modules/modulefiles/test sudo tee /usr/share/modules/modulefiles/test/1.0 << 'EOF' #%Module1.0 proc ModulesHelp {} { puts stderr "Módulo test - binario en /opt/apps/testbin" } module-whatis "Ejecutable test.x" prepend-path PATH /opt/apps/testbin EOF

Copiar modulefile a los nodos de cómputo:

for i in 1 2 3; do scp -r /usr/share/modules/modulefiles/test root@172.16.0.1$i:/usr/share/modules/modulefiles/ done

🚀 9. Lanzar trabajo con Slurm (partición outes2)

su - sebas1 # importante el guión para ir a su home cat > ~/script.sh << 'EOF' #!/bin/bash #SBATCH --job-name=test_modules #SBATCH --output=/home/sebas1/salida #SBATCH --partition=outes2 source /etc/profile.d/modules.sh module load test/1.0 test.x EOF sbatch ~/script.sh sleep 8 cat ~/salida # Salida esperada: "hola sebas1" + fecha actual
✅ El trabajo se ejecutará en computer-0-2 o computer-0-3. Verificar con squeue y sinfo.

🛠️ Solución de problemas comunes

  • munge: Header lengths → chrony + regenerar clave munge.
  • Nodos Slurm DOWN/DRAIN → scontrol update nodename=computer-0-X state=RESUME o limpiar spool.
  • RealMemory mismatch → ejecutar slurmd -C en cada nodo y ajustar slurm.conf.
  • autofs no monta /opt/apps → montaje vía fstab como se indica arriba.
  • getent passwd sin usuarios LDAP → verificar /etc/nsswitch.conf, /etc/hosts y reiniciar nscd.
